# 手動テストチェックリスト

各項目を three.js / p5.js / 生WebGL の3系統で確認する。
（実行: `npm run dev` でアプリを起動し、URLバーに作品URLを入れて「読込」）

## 表示・操作
- [ ] URLを読み込むと作品が枠内に表示される
- [ ] 作品をマウス操作できる（ドラッグ・クリック・スクロール）
- [ ] 比率を切り替えると枠と作品が即座に組み直される
- [ ] 任意W:H（例 21:9）を適用すると枠がその比率になる
- [ ] ウィンドウをリサイズしても枠が追従し、はみ出さない

## 枠＝撮影枠の一致
- [ ] 静止画の出力範囲が、画面で見えていた枠と一致する
- [ ] 動画の出力範囲が、画面で見えていた枠と一致する
  - ※ 動画はアプリウィンドウを desktopCapturer で取得しクロップしている。
    OSのタイトルバー分のオフセットや、ウィンドウ以外のソースが選ばれた場合に
    ズレが出ないかを特に確認する。ズレる場合は `recorder.ts` のクロップ原点と
    `displayMedia.ts` のソース選択を実機に合わせて調整する。

## 静止画
- [ ] 長辺px指定で、出力PNGがその実ピクセルになる（`sips -g pixelWidth -g pixelHeight <file>` で確認）
- [ ] cm+dpi指定で実寸どおりのピクセルになる（10cm/300dpi≒1181px）
- [ ] 高DPRでも構図が崩れない（拡大ではなく高精細になっている）
  - ※ `deviceScaleFactor` を上げた capturePage が実際に高解像度PNGを返すかを確認する。
    もし解像度が上がらない場合は capturePage の挙動を実機で再検証する（設計上の最大リスク点）。
- [ ] 16384px超の指定で警告トーストが出て上限に丸められる
- [ ] 1:1 で正方形PNGが出る
- [ ] 保存ダイアログで保存先を選べる

## 動画: Render mode（オフラインレンダリング）
- [ ] Render+2160で重い作品を録ってもフレーム数=秒数×60
      （`ffprobe -select_streams v:0 -show_entries stream=width,height,r_frame_rate,nb_frames`で確認）
- [ ] 録画開始でページが**リロードされない**（作品にセットしたパラメータが録画前後で保持される）
- [ ] 録画中はフリーズ画像+進捗バー、Cancelで中断できる（ファイルは保存されない）
- [ ] 最終フレーム後は「Finalizing…」表示になりボタンが無効化される
- [ ] 録画終了/中断後にプレビューが実時間動作へ戻る
- [ ] rAFベースでない作品（CSSアニメ・`<video>`等）は進まない=既知の制約
- [ ] 画面表示より大きいプリセットでも実レンダリングされる（引き伸ばしでなく高精細）
- [ ] Motion blur 4xで移動体に自然な尾引きが出る（レンダリング時間は約4倍）
- [ ] Supersample 2xでエッジのジャギーが減る
- [ ] mp4にBT.709の色タグが付く（ffprobeでcolor_space/primaries/transfer=bt709）

## 静止画: 品質オプション
- [ ] Supersample 2xでエッジのジャギーが減る
- [ ] cm/dpi指定のPNGにpHYs（指定DPI）とsRGBチャンクが入る（px指定ではpHYs無し）
- [ ] PhotoshopでPNGを開くと指定した実寸/DPIになっている

## 動画: Live mode（画面録画）
- [ ] 操作しながら録れる（カーソル込み・滑らかさを作品ごとに記録）
- [ ] システム音声/入力デバイス音声が入る（ScreenCaptureKit対応環境）
- [ ] 音声非対応環境では警告トーストが出て映像のみで保存される
- [ ] 画面表示より大きいプリセットはキャップされ、トーストで実解像度を知らせる
- [ ] webm保存→mp4/WebP変換が成功する

## エラー
- [ ] 不正URL（例 `https://invalid.invalid`）で読込失敗トーストが出てもアプリは生存する
- [ ] 保存ダイアログのキャンセルで何も起きない（エラートーストも出ない）
